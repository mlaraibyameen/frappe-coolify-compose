# Contributing

Thank you for your interest in contributing! This is a community resource and improvements are very welcome.

## Ways to contribute

- **Report bugs** — open an [issue](../../issues) with steps to reproduce
- **Suggest improvements** — open an issue describing the use case
- **Submit a PR** — see below for guidelines
- **Improve docs** — typos, clarifications, and better examples are always welcome

## Development setup

1. Fork and clone the repo
2. Create a branch: `git checkout -b fix/my-fix` or `feat/my-feature`
3. Make your changes
4. Open a pull request against `main`

## Pull request guidelines

- Keep PRs focused — one change per PR
- Update `.env.example` and `README.md` if you add new environment variables
- Test your changes against a real Coolify instance if possible
- Describe _why_ you made the change, not just what changed

## Reporting issues

Please include:
- Coolify version
- Frappe/ERPNext/HRMS image version (`VERSION` env var)
- The service that failed and its logs (copy from Coolify)
- What you expected vs what happened

## Code of conduct

Be respectful. This project follows the [Contributor Covenant](https://www.contributor-covenant.org/).
